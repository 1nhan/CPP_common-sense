"Hello, World!" 프로그램의 목적은 무엇인가요?
<details><summary>답변 보기</summary>
"Hello, World!" 프로그램은 프로그래밍 언어의 기본 구조를 테스트하고, 
개발 환경이 제대로 설정되었는지 확인하기 위한 가장 간단한 예제입니다. 
주로 입문자들이 첫 번째로 작성하는 프로그램이며, 
출력 기능과 컴파일 과정을 이해하는 데 도움을 줍니다.
</details>

함수의 네 가지 구성 요소를 말해보세요.
<details><summary>답변 보기</summary>
1. 반환형 (Return Type)
2. 함수 이름 (Function Name)
3. 매개변수 목록 (Parameter List)
4. 함수 본문 (Function Body)
</details>

모든 C++ 프로그램에 반드시 포함되어야 하는 함수는 무엇인가요?
<details><summary>답변 보기</summary>`main()` 함수입니다. 
C++ 프로그램은 실행 시 `main()` 함수부터 시작되며, 
이 함수가 없으면 프로그램이 실행되지 않습니다.
</details>

"Hello, World!" 프로그램에서 return 0;의 목적은 무엇인가요?
<details><summary>답변 보기</summary>
`return 0;`은 프로그램이 정상적으로 종료되었음을 운영체제에 알리는 역할을 합니다. 
0은 일반적으로 성공을 의미합니다.</details>

컴파일러의 역할은 무엇인가요?
<details><summary>답변 보기</summary>
컴파일러는 사람이 작성한 소스 코드(C++)를 기계가 이해할 수 있는 
오브젝트 코드(기계어)로 변환하는 프로그램입니다.</details>

import 문장의 목적은 무엇인가요?
<details><summary>답변 보기</summary>
`import` 문은 다른 모듈이나 라이브러리의 기능을 현재 코드에서 사용할 수 있도록 
불러오는 역할을 합니다. C++에서는 주로 `module` 기능과 함께 사용됩니다.
</details>

#include 지시문의 목적은 무엇인가요?
<details><summary>답변 보기</summary>
`#include`는 헤더 파일이나 다른 소스 파일의 내용을 현재 파일에 포함시키는 
전처리 지시문입니다. 
예: `#include <iostream>`은 입출력 기능을 사용하기 위해 필요합니다.
</details>

파일 이름 끝의 .cpp 확장자는 C++에서 무엇을 의미하나요?
<details><summary>답변 보기</summary>
`.cpp` 확장자는 해당 파일이 C++ 소스 코드 파일임을 나타냅니다. 
컴파일러는 이 확장자를 통해 C++ 문법으로 해석합니다.
</details>

링커(linker)는 프로그램에서 어떤 역할을 하나요?
<details><summary>답변 보기</summary>
링커는 여러 오브젝트 파일들을 하나의 실행 파일로 결합하며, 
함수나 변수의 참조를 해결해줍니다.
</details>

소스 파일(source file)과 오브젝트 파일(object file)의 차이는 무엇인가요?
<details><summary>답변 보기</summary>
소스 파일: 사람이 작성한 코드 (.cpp 등)
오브젝트 파일: 컴파일된 중간 결과물 (.obj 또는 .o 등)
</details>

실행 파일(executable)이란 무엇인가요?
<details><summary>답변 보기</summary>
실행 파일은 컴퓨터가 직접 실행할 수 있는 기계어로 구성된 파일입니다. 
보통 `.exe` 확장자를 가집니다.
</details>

IDE란 무엇이며, 어떤 기능을 제공하나요?
<details><summary>답변 보기</summary>
IDE(통합 개발 환경)는 코드 작성, 디버깅, 컴파일, 실행 등을 하나의 환경에서 
할 수 있도록 도와주는 소프트웨어입니다. 예: 
Visual Studio, Code::Blocks
</details>

컴파일된 프로그램을 실행하려면 어떻게 해야 하나요?
<details><summary>답변 보기</summary>
컴파일 후 생성된 실행 파일(.exe)을 더블 클릭하거나 명령어를 통해 실행하면 됩니다. 
예: `./program.exe`
</details>

주석(comment)이란 무엇인가요?
<details><summary>답변 보기</summary>
주석은 코드에 대한 설명을 작성하는 부분으로, 컴파일러는 이를 무시합니다. 
예: `// 한 줄 주석`, `/* 여러 줄 주석 */`
</details>

drill의 목적은 무엇인가요?
<details><summary>답변 보기</summary>
drill은 반복적인 연습을 통해 개념을 확실히 익히고, 
프로그래밍 실력을 향상시키기 위한 학습 방법입니다.
</details>
